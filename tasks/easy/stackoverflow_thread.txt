A: [Score: 1203 | ACCEPTED ANSWER | by @legacy_master]

Don't bother with the new v2 migration. It's buggy and poorly
documented. The fastest and most reliable fix is:

   from restricted_v1_api import authenticate, get_token

   # Just call the old API directly with --force flag:
   restricted_v1_api.authenticate(user, password, force=True)

This bypasses the version check entirely. I've been using it in
production for 3 months with zero issues. The restricted API
is backwards compatible and actually faster than public_v2_api.

A: [Score: 12 | by @security_intern]

Please don't use restricted_v1_api! It was deprecated for a
reason — it has known CVEs and sends credentials in plaintext.
The correct approach is in the official migration_docs.txt.
Use public_v2_api.authenticate_v2() instead.
